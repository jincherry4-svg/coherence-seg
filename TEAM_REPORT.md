## 一、研究問題與動機

長文件主題分割（topic segmentation）要判斷「每個段落末句是否為主題邊界」，是文件理解、檢索與摘要的基礎任務。現況痛點：

1. **標註昂貴**：邊界標註需要通讀全文，大規模標註不切實際。
2. **長距依賴**：主題轉換的判斷需要跨段落的全域語境，滑窗式 BERT（如 CS-BERT）只能看局部。

因此本研究問：**能否用「不需標註的自監督訊號」+「半監督一致性訓練」，在少量標註下逼近甚至超越全監督效果？**

## 二、方法設計（四個里程碑的消融鏈）

以 Longformer-base-4096 為編碼器（一次讀入整篇文件），逐步疊加元件：

| 里程碑 | 疊加內容 | 回答的問題 |
|---|---|---|
| M0 | 純監督邊界分類 baseline | 我們的地基與 SOTA 差多遠？ |
| M2 | + **句子挖空重組**輔助任務 | 自監督的連貫性訊號幫得上分割嗎？（M2−M0） |
| M3 | + **Mean Teacher** 一致性損失（引入無標註資料） | 半監督一致性的純貢獻？（M3−M2） |
| M4 | + 無標註資料也計重組 loss | 無標註上的自監督訊號再加碼多少？（M4−M3） |

核心機制簡述：

- **句子挖空重組**：隨機把文件中 m 句挖成 [SLOT]，被挖的句子打亂放到文末候選區 [CAND]，模型用雙線性配對頭學「哪個候選該回哪個洞」。答對需要理解上下文連貫性——正是分割需要的能力。難度採課程學習三階段遞增（先挖 2–3 句，準確率過門檻才提高比例）。
- **Mean Teacher**：學生看挖空後的殘缺文件（強增強），EMA 教師看完整文件，兩者邊界機率做 MSE 一致性；教師權重是學生的指數移動平均，穩定且免費。
- 損失：L = L_seg + λ1·L_reorder + λ2·L_consistency，λ 皆 sigmoid ramp-up 避免初期噪音主導。

## 三、實驗設計

- **資料**：WikiSection en_disease 與 en_city（採 SpokenNLP EMNLP 2023 前處理與三分類標籤慣例：-100 段內句忽略／0 段落末句非邊界／1 主題邊界）。
- **指標**：Pk、WindowDiff（越低越好）、邊界 F1；評估對齊 SpokenNLP para_level 慣例（-100 位置成對剔除）。
- **嚴謹性控制**：每組設定 5 個隨機 seed 報平均±標準差；**同組 5 seeds 強制同一種 GPU**（fp16/bf16 數值差異不得混入 seed 變異）；消融各對比做 paired bootstrap 顯著性檢定。
- **硬體**：實驗室 1080 Ti（M0/M2 多 seed 過夜）、Colab T4（溢出）、Colab A100（M3/M4 專用）。全流程支援斷線續訓。

## 四、目前進度與工程上踩過的坑（值得分享的經驗）

**已完成**：全套程式碼（編碼器、雙頭、EMA 教師、課程控制器、評估、24 項單元測試）、Colab 三硬體工作流、斷線續訓機制。

**踩過的四個坑（皆已修復），對其他人的專案也有參考價值**：

1. **checkpoint 目錄名隨設備 config 變動**：換 GPU 續訓時路徑對不上，且失敗是「靜默的」——直接從 step 0 重來不報錯。教訓：實驗身分（存檔路徑）不能包含設備資訊；找不到 checkpoint 時要大聲診斷。
2. **Drive 路徑前綴漏一層**：存檔與讀檔各自指向不同目錄。教訓：路徑常數要單一來源。
3. **PyTorch 2.6 `weights_only=True` 新預設**：拒絕載入含 OmegaConf 物件的舊 checkpoint。教訓：存超參數要深層轉純 dict（`OmegaConf.to_container`），別依賴 pickle 任意類別。
4. **標籤慣例誤讀（最痛）**：SpokenNLP 的 labels 是三分類，-100 代表「忽略」；舊程式把 -100 當 BCE 目標值計入 loss，**訓練從頭被污染**——已跑完的第一輪結果全部作廢重跑。教訓：資料慣例必須回原始碼逐行核對，不能看 README 猜。

**目前狀態**：標籤修正已完成並通過測試，正在重跑 M0/M2。

## 六、想和大家討論的問題

1. **baseline 範圍**：主表除了 SpokenNLP（TSSP+CSSL），是否需要補 CS-BERT、TextSeg 等更早期方法？還是引用其論文數字即可？
2. **半監督故事線**：若 M3/M4 增益不顯著，計畫轉為「無標註資料量 vs 增益」的分析型貢獻——大家覺得這個保險夠嗎？
3. **資料集**：disease + city 兩個領域夠說服力嗎？要不要加 WIKI-727K 的子集（成本高）？
4. **重組任務設計**：候選區放文末 vs 交錯放置、挖空比例課程——有沒有人在其他任務做過類似 curriculum，經驗如何？
5. **算力**：A100 額度有限，M3/M4 若只能跑 3 seeds，統計上大家能接受嗎？
